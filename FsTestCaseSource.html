Using TestCaseSource with FsUnit in Xamarin

Recently, I attended my first F# meetup here in Japan. Actually, they don't call it a meetup. The term in Japanese is F#談話室 which roughly translates to F# conversation room. The atmosphere was pretty laid back. Many members seemed to know each other. The group was small compared to some of the other cities’ F# meetups, but it gave the gathering an intimate feeling, The attendees included F# MVPs, the translator of a number of F# programming books, and the author of the open source FSRandom library. In case you didn't know, this is also the group that came up with the idea for the F# Advent Calendar.

Since this was my first meeting, I decided to give a 'spur of the moment' lightning talk on my experience using the <TestCaseSouree> Attribut with FsUnit. The talk focused on the advantages of using it for certain cases of TDD.

Specifically, I found this approach useful where you have the following scenario

  • Pass in some state to a function
  • Pass in something that will modifiy that state
  • Return the modified state and accumulate a list of changes in the state

This approach is very useful for testing a sequence events by recursively iterate over a list of data events accumulating response

module SimpleExample = 

    let modifyStateAndRtnChgs state modifier  =  
        let newState = state * (1. + modifier)
        newState,  newState - state

    let ms = [-0.03; -0.02; 0.03; 0.021] // a list of data points: down -3%, -2%, then back up +3%, +2.1%  
     
    let rs = ms |> List.scan (fun (state, _) modifier -> modifyStateAndRtnChgs state modifier) (1., 0.)

The List.scan operation builds a list of results by iterating over a list data points and accumulating the results. This nicely simulates feeding in a series of events and getting back a series of results.

The results can then be parsed using simple list operations.

Here how you can combine the above with TestCaseSource attribute to create some consise unit tests.

[<TestFixture>]
type ``A Sample Test`` () = 

    let approxMatch ls1 ls2 e =
       List.forall2 (fun (x1:float) (x2:float) -> Math.Abs(x1 - x2) <= e) ls1 ls2

FsUnit has a great set of comparision functions but you may need to write your own depending on what you are testing, Here we add an approxMatch function which compares two lists of values to see if they are an approximate match

    static member TestData = 
        [|
          [|[-0.01; -0.02; 0.02; 0.01 ], [0.0; -0.01; -0.0198; 0.019404; 0.00989604 ]|]; // test 1
          [|[-0.03; -0.02; 0.03; 0.021], [0.0; -0.03; -0.0194; 0.028518; 0.020561478]|]; // test 2
        |]

There are a few important things ot note about the smaple data.

  1. Nunit expects it to take the form of a static member
  2. It is defined as an array of arrays ie. [ ][ ]
  3. You can put almost any object in the inner nested array. In this case it is a tuple (list * list)
  4. This two item tuple is a simple way to keep your function inputs and outputs clear

    [<TestCaseSource("TestData")>]
    member x.``Resulting list has the correct length`` (testData: float list * float list) = 
        
        let data, expectedResults = testData
        let states, results = 
            data |> List.scan (fun (s, _) m -> SimpleExample.modifyStateAndRtnChgs s m) (1., 0.) 
            |> List.unzip
        
        results |> should haveLength expectedResults.Length


    [<TestCaseSource("TestData")>]
    member x.``List result's approximiately match expected results`` (testData: float list * float list) = 
        
        let data, expectedResults = testData
        let states, results = 
            data |> List.scan (fun (s, _) m -> SimpleExample.modifyStateAndRtnChgs s m) (1., 0.) 
            |> List.unzip
        
        approxMatch results expectedResults 1e-9 |> should be True
