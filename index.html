Minds, Markets, and Machines

Welcome to Minds, Markets and Machines

Wherein, we take a casual stroll through the intersection of machine learning, quantitative analysis, and functional programming. I work in the field of what is now often called 'alpha discovery'. If you're not sure what that means, just think data science for finance.

Like data science in general, it encompasses a number of domains:

  • Math
  • Statistics analysis
  • Machine learning
  • Diverse programming skills

It goes a bit further. To do it really well, you really need an understanding of

  • Behavioral economics
  • Market structure (financial domain knowledge)

Prepare to be tested (aka “Your pedigree means nothing”)

It often feels like Kaggle only with more painful feedback. If you model doesn’t work out, you get a slap in the 'pnl face' from the market. Oh... and there’s plenty of feedback to go around.

Actionable decisions from distant feedback

Perhaps most importantly, it is about creating models that produce immediately actionable decisions where the feedback is distant in time from the inputs used to make those decisions. In that sense, it seems closer to robotics than data science.

Wait... What's functional programming got to do with it?

Functional programming in uniquely suited to the task. There is of course all the stuff you hear about functional programming that is true for any domain.

  • Clean, concise code leading to fewer bugs and easier maintenance
  • Immutability by default, which prevents a large class of errors.
  • Concurrency - asynchronous programming is very easy, as is parallelism.
  • Domain modelling is very easy and explicit

But there is one aspect that is seems under appreciated in my field.
